Ameto observability

Ameto is a self-hosted observability backend for logs, traces, metrics and alerts — one place to search, correlate and be paged.

LogsStructured events (CLEF). Full Seq-style filter language with 40+ functions.
TracesDistributed spans. Waterfall, service graph, latency + TraceQL search.
MetricsCounters, gauges & histograms. Aggregations, group-by, exemplars.
AlertsRules over any signal, routed to Slack / Teams / PagerDuty / webhooks.

Data is correlated by @tr (trace id) and @sp (span id): jump from a log line to its trace, from a trace span to its logs, and from a metric exemplar into the originating trace. Ingest is CLEF over HTTP (/api/events, Seq-compatible) and OTLP.

Logs & the filter language

The search box takes a filter expression. Bare words are a free-text search; anything with a field, operator or function is a structured clause. String comparisons are case-insensitive; a comparison against an array property matches if any element matches.

Property paths

Nested props use dots; non-identifier keys and array indices use brackets.

  • Payload.Customer.Id — nested object
  • Headers['Api-Request-Id'] — key with hyphens/dots
  • Tags[0] — array element by index

Traces & TraceQL

The Traces page has structured filters (service / name / status / duration / HTTP) and a TraceQL mode for attribute search. A query is a span-set filter in braces: { … }. All conditions inside the braces must hold on the same span; a trace is returned if any of its spans matches.

Attributes need a leading dot: .db.system = 'mssql', not db.system. Intrinsics (below) are written bare.

Alerts

Alert rules (the Alerts page) evaluate a condition over logs, metrics or traces on a schedule and dispatch a notification when it fires — with re-notify, acknowledge, escalation, severity routing and maintenance windows.

  • Channels: Slack, Discord, MS Teams, PagerDuty, and a Postman-style HTTP-flow (multi-step requests with variable capture).
  • Secrets (bot tokens, webhook auth) are stored encrypted (AES-256-GCM) and masked in the API.
  • Silences & maintenance windows mute dispatch for a day/time range and severity scope.
